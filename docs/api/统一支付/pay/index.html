    统一支付

    说明#

    此种支付会返回一个支付URL，需接入端主动渲染

    时序说明#

    请求URL#

    /api/order

    请求方式#

    POST请求，Content-Type为application/json

    请求字段#

    字段类型长度示例可否为空说明
    appidstring8-32"test1234"否商户接入appid，需创建商户申请
    merchantTradeNumberstring1-64---否商户方单号
    amountLong0-12---否金额(THB),精确到分，例如1THB,需传100
    timestampLong12-20---否下单时间戳，10分钟内下单有效
    notifyUrlstring12-20---否异步回调地址
    successRedirectUrlstring------是支付成功后，用户页面回跳 URL 地址
    failRedirectUrlstring------是支付失败后，用户页面回跳 URL 地址
    paymentMethodIdlong101否支付方式ID，可在获取支付方式的接口中获取对应的值
    paymentMethodstring20"alipay"否支付方式，可在获取支付方式的接口中获取对应的值
    storeIdstring20"12389182317371"是线下商户必填（未说明，则为线上用户）
    remarkstring128---传值则需要签名，不传不需要签名备注
    remark2string-----传值则需要签名，不传不需要签名备注2
    remark3string-----传值则需要签名，不传不需要签名备注3
    signstring12-20---否签名串

    请求示例#

    {    "appid":"test1",    "merchantTradeNumber":"12345678920100005",    "amount":"100",    "paymentMethodId":1,    "paymentMethod":"alipay",    "timestamp":"1630482083910",    "remark":"外卖订单",    "sign":"c9b90fa7113bd458ed9ff9352759d49b1b07f80c5301790aa856ed41ea0a6eb8"   }

    返回内容说明#

    字段类型长度示例可否为空说明
    codeint---200否业务码 200 为成功下单
    msgstring---"ok"否对业务码说明
    datadata-----否对象参数

    data参数说明#

    字段类型长度示例可否为空说明
    tradeNumberstring---"1432972000019742720"否单号
    merchantTradeNumberstring---"12345678920100005"否商户单号
    payUrlstring---"ok"否支付链接
    payChannelstring---"OP"否接口服务标示
    paymentMethodstring---"alipay"否支付方式
    createTimestring---"1630482122794"否创建时间
    statusstring---"PAYING"否订单状态，具体参考状态说明

    返回内容示例#

    {    "msg": "ok",    "code": 200,    "data": {        "tradeNumber": "1432972000019742720",        "merchantTradeNumber": "12345678920100005",        "payUrl": "https://*****.com",        "payChannel": "OP",        "paymentMethod": "alipay",        "createTime": 1630482122794,        "status": "PAYING",        "amount": 100    }}
